# ECE410 Project — LUT Kernel SVM Compute Core (m4: 128-Feature Hardened)

5-class cardiac arrhythmia detection (Normal / PVC / AFib / VT / SVT) using a
fixed-point RBF-SVM accelerator with a range-reduction LUT kernel (γ = 0.25,
128-dim single-beat features, Q6.10 arithmetic).

> **m4 parameters:** `FEATURE_DIM=128  NUM_SV=256  FIFO_DEPTH=4096  ADDR_WIDTH=12`
> Instantiated at synthesis via `chparam -set FEATURE_DIM 128 -set NUM_SV 256`.
> Structural testbenches (error codes, backpressure, distance, etc.) are
> parameter-agnostic and pass with any `FEATURE_DIM`. The full-pipeline
> classifier test (`tb_svm_classifier.sv`) uses hex data generated for the
> 256-dim model; regenerate `tb_svm_params.svh` with `gen_tb_data.py
> --feature_dim 128 --num_sv 256` for an exact m4 reference run.

## Register Map

All configuration is written through the `param_write_en` / `param_addr` /
`param_data` interface. Values are Q6.10 fixed-point (multiply float by 1024
and round to the nearest integer).

| `param_addr` | Register | Reset value | Float | Notes |
|-------------|----------|-------------|-------|-------|
| `3'b000` | `gamma_reg` | `0x0100` | 0.25 | RBF kernel bandwidth |
| `3'b001` | `c_reg` | `0x0400` | 1.0 | SVM regularisation (OvO, host-side) |
| `3'b010` | `bias_reg[0]` | `0x0000` | 0.0 | Decision bias — Normal |
| `3'b011` | `bias_reg[1]` | `0x0000` | 0.0 | Decision bias — PVC |
| `3'b100` | `bias_reg[2]` | `0x0000` | 0.0 | Decision bias — AFib |
| `3'b101` | `bias_reg[3]` | `0x0000` | 0.0 | Decision bias — VT |
| `3'b110` | `bias_reg[4]` | `0x0000` | 0.0 | Decision bias — SVT |
| `3'b111` | *(reserved)* | — | — | Writes silently ignored |

Registers are latched on the `start` pulse and held fixed for the duration of
each batch.

### Bias Registers

`bias_reg[0..4]` apply a per-class additive offset to each class's OvO
decision score. They are exposed as Q6.10 outputs and are intended to be
consumed by the host MCU alongside the kernel results to shift decision
boundaries — for example, to trade sensitivity against specificity for a
particular arrhythmia class without retraining.

**Example — shift the PVC boundary by +0.5:**

```
param_write_en = 1
param_addr     = 3'b011       // bias_reg[1] = PVC
param_data     = 16'h0200     // +0.5 in Q6.10 (0.5 × 1024 = 512 = 0x0200)
```

Default reset value is `0x0000` (no offset) for all five classes.

## Classes and SV Counts

| Index | Class | Full-model SVs | HW-capped SVs (`MAX_SV_PER_CLASS = 50`) |
|-------|-------|---------------|----------------------------------------|
| 0 | Normal | 45 | 45 |
| 1 | PVC | 157 | 50 |
| 2 | AFib | 56 | 50 |
| 3 | VT | 41 | 41 |
| 4 | SVT | 178 | 50 |
| | **Total** | **477** | **236** |

`num_sv_per_class[0..4]` must be programmed before asserting `start`. The
hardware validates that `0 < Σ sv_count ≤ NUM_SV`; if the check fails the
`error` flag is asserted and the FSM stalls in `ERROR_STATE`.

## Diagnostic Error Codes

The core exposes two error outputs:

| Port | Width | Behaviour |
|------|-------|-----------|
| `error` | 1-bit | Asserts high on first fault; holds until `rst_n` |
| `error_code` | 4-bit | Latches code of first fault; holds until `rst_n` |

Both are **sticky** — they do not clear on `start` and do not update if a
second fault occurs. The only way to clear them is to assert `rst_n` low for
at least four clock cycles.

| Code | Constant | Priority | Condition | MCU action |
|------|----------|----------|-----------|------------|
| `0x0` | `ERR_NONE` | — | No fault; normal operation | — |
| `0x1` | `ERR_SV_ZERO` | Highest | `Σ sv_count = 0` when FSM leaves `IDLE` — no SVs programmed before `start` | Re-program `num_sv_per_class`, assert `rst_n` |
| `0x2` | `ERR_SV_OVERFLOW` | ↑ | `Σ sv_count > NUM_SV` — hardware capacity exceeded | Reduce per-class SV counts, assert `rst_n` |
| `0x3` | `ERR_ILLEGAL_STATE` | ↑ | FSM reached `default` branch — undefined state transition | Assert `rst_n`; inspect waveform |
| `0x4` | `ERR_GAMMA_SAT` | ↓ | `gamma_int > 0x2000` (γ > 8.0) — LUT returns 0 for all I ≥ 8, making every kernel output zero | Reprogram `gamma_reg` to ≤ `0x2000`, assert `rst_n` |
| `0x5` | `ERR_FIFO_OVERFLOW` | ↓ | `fifo_full && qspi_valid` during `LOAD_FIFO` — one or more feature words dropped | Reduce QSPI clock rate or increase `FIFO_DEPTH`; assert `rst_n` |
| `0x6` | `ERR_GAMMA_ZERO` | Lowest | `gamma_int == 0` while FSM not idle — all kernels silently return 1.0 (constant classifier) | Reprogram `gamma_reg` to a non-zero value, assert `rst_n` |
| `0x7–0xF` | *(reserved)* | — | Reserved for future extension | — |

> **Note on `ERR_GAMMA_SAT`:** γ = 0.25 (`0x0100`) is the trained value and
> is safe. The threshold of 8.0 (`0x2000`) is conservative — in practice any
> γ that causes `I ≥ 8` for most support-vector distances will collapse
> classifier accuracy. See `horner_lut_math.pdf` Section 7 for the full
> analysis.

## Power Budget (Wearable Strap-On Detector)

> All figures assume a **custom ASIC** implementation at 180 nm.
> An FPGA prototype (e.g. Lattice iCE40) draws 5–20× more and is not
> suitable for a battery-powered wearable.

### Operating Modes

**Monitor Mode** — continuous, runs 99.8 %+ of the time.
ECG is sampled at 360 Hz; the MCU runs lightweight R-peak detection and
accumulates RR/waveform buffers. The SVM core is clock-gated.

**Compute Mode** — burst, triggered once per heartbeat (~857 ms at 70 bpm).
Duration per beat: 256 µs QSPI transfer + ~1.3 ms SVM kernel (236 SVs ×
275 cycles @ 50 MHz) + ~50 µs MCU OvO ≈ **1.6 ms per beat**.
Duty cycle at 70 bpm: 1.6 ms / 857 ms ≈ **0.19 %**.

### Per-Component Power Draw

| Component | Monitor Mode | Compute Mode | Notes |
|-----------|-------------|-------------|-------|
| ECG analog front end | 1.0 mW | 1.0 mW | 300 µA @ 3.3 V (e.g. TI ADS1291) |
| Host MCU | 0.5 mW | 3.0 mW | Low-power run (4 MHz) vs. active OvO |
| SVM ASIC core | 0.5 mW | 10.0 mW | Clock-gated idle vs. 50 MHz active |
| Off-chip SRAM (SV + WS) | 0.3 mW | 3.3 mW | Standby vs. burst read/write |
| **Subtotal** | **2.3 mW** | **17.3 mW** | |
| BLE radio *(optional)* | +0.5 mW | +0.5 mW | Alert-only advertising |

### Average Power

```
P_avg = (0.9981 × 2.3 mW) + (0.0019 × 17.3 mW)
      ≈ 2.30 + 0.03
      ≈ 2.33 mW  (2.83 mW with BLE)
```

Average draw from a 3.7 V LiPo (assuming 85 % regulator efficiency):

| Configuration | Avg current from battery |
|---------------|--------------------------|
| Without BLE | **0.74 mA** |
| With BLE alerts | **0.90 mA** |

### Battery Sizing

| Battery | Capacity | Without BLE | With BLE |
|---------|----------|-------------|----------|
| CR2032 coin cell | 240 mAh | ~13.5 days | ~11 days |
| 150 mAh LiPo | 150 mAh | ~8.5 days | ~7 days |
| 200 mAh LiPo | 200 mAh | ~11.2 days | ~9.3 days |
| 300 mAh LiPo | 300 mAh | ~16.9 days | ~13.9 days |

*Runtime = (capacity × 0.85) / avg\_current*

### Recommendation

A **200–300 mAh LiPo** (roughly the size of a stack of 6–8 credit cards,
~3 × 5 × 0.5 cm) balances runtime against form factor for a chest or wrist
strap:

- **200 mAh** → ~11 days between charges without BLE (~9 with BLE alerts).
  Comparable to the Zio Patch extended Holter (14-day target, 180 mAh).
- **300 mAh** → ~17 days without BLE. Suitable for post-discharge cardiac
  monitoring where the patient cannot charge daily.

> **FPGA prototype note:** A Lattice iCE40HX running the same RTL draws
> ~50–150 mW in compute mode and ~20–40 mW idle, giving only 1–4 hours on
> a 200 mAh cell. Use the FPGA only for bench validation; tape out to ASIC
> before wearable deployment.

## Multi-Scale Feature Vector (256 dims, Q6.10)

| Dims | Source | How computed |
|------|--------|-------------|
| `[0:128]` | Single-beat morphology | 128 samples centred on R-peak (±64), amplitude-normalised to [−1, 1] |
| `[128:192]` | 10-beat mean morphology | Mean of 10 × 64-sample snippets (previous beats), amplitude-normalised to [−1, 1] |
| `[192:256]` | 100-beat RR track (causal) | 99 RR intervals from last 100 beats ÷ NORMAL\_RR (308), clipped to [0, 2], resampled to 64 points |

The RR track is **causal** — it uses only past beats, so no lookahead buffer is
needed on the host MCU beyond a ring buffer of 100 R-peak timestamps (~200 B).

## Pre-Netlist Verification — Results

All 19 tests (10 iverilog + 9 cocotb) pass against the fixed RTL.
See `ECE_project_tb_netlist/tb_results.md` for the full results table.

### RTL fixes applied

| Issue | Testbench | Severity | Fix applied |
|-------|-----------|----------|-------------|
| `kernel_valid` held for only 1 cycle — any late `kernel_ready` stalled the FSM permanently | `tb_backpressure.sv` sub-test C | **High** | Set/clear latch: `kernel_valid` holds until `kernel_ready && kernel_valid` handshake |
| `gamma_int` updated immediately on `param_write_en` — mid-compute write corrupted in-flight kernels | `tb_param_write.sv` | **Medium** | `gamma_latched` shadow register captured at `start`, wired to Horner engine |
| `gamma = 0` produced `kernel_out = 1024` for every SV with no error flag | `tb_gamma_zero.sv` | **Low** | Added `ERR_GAMMA_ZERO (0x6)`: asserts when `gamma_int == 0 && state != IDLE` |

# ECE410 Project — LUT Kernel SVM Compute Core (Pre-Netlist Verified)

5-class cardiac arrhythmia detection (Normal / PVC / AFib / VT / SVT) using a
fixed-point RBF-SVM accelerator with a range-reduction LUT kernel (γ = 0.25,
256-dim multi-scale features, Q6.10 arithmetic).

This folder contains the **verified pre-netlist RTL** with all testbench-driven
bugs fixed and a complete 19-test suite (10 iverilog + 9 cocotb) passing.

---

## RTL Fixes Applied

Three bugs were discovered and fixed by running the pre-netlist testbench suite.

| # | Fix | Root cause | Testbench that found it |
|---|-----|-----------|------------------------|
| 1 | **`kernel_valid` hold register** — changed from a 1-cycle pulse to a set/clear latch held until `kernel_ready && kernel_valid` handshake | `kernel_valid` tracked `horner_valid_out` for exactly one cycle; any late `kernel_ready` left the FSM stalled in `OUTPUT_RESULT` permanently | `tb_backpressure.sv` sub-test C |
| 2 | **Gamma shadow register** (`gamma_latched`) — `gamma_int` is now latched at the `start` pulse and `gamma_latched` is wired to the Horner engine for the duration of the batch | `gamma_int` updated immediately on `param_write_en` regardless of FSM state; a mid-compute write corrupted in-flight kernel values | `tb_param_write.sv` |
| 3 | **`ERR_GAMMA_ZERO` (code `0x6`)** — asserts when `gamma_int == 0` while the FSM is not idle | γ = 0 makes P = 0 for all distances, so every kernel silently returns 1.0; no error was raised | `tb_gamma_zero.sv` |

## Register Map

Values are Q6.10 fixed-point (multiply float by 1024 and round).

| `param_addr` | Register | Reset value | Float | Notes |
|-------------|----------|-------------|-------|-------|
| `3'b000` | `gamma_reg` | `0x0100` | 0.25 | RBF kernel bandwidth |
| `3'b001` | `c_reg` | `0x0400` | 1.0 | SVM regularisation (host-side OvO) |
| `3'b010` | `bias_reg[0]` | `0x0000` | 0.0 | Decision bias — Normal |
| `3'b011` | `bias_reg[1]` | `0x0000` | 0.0 | Decision bias — PVC |
| `3'b100` | `bias_reg[2]` | `0x0000` | 0.0 | Decision bias — AFib |
| `3'b101` | `bias_reg[3]` | `0x0000` | 0.0 | Decision bias — VT |
| `3'b110` | `bias_reg[4]` | `0x0000` | 0.0 | Decision bias — SVT |
| `3'b111` | *(reserved)* | — | — | Writes silently ignored |

`gamma_reg` is **latched into a shadow register** at the `start` pulse. A
`param_write_en` write to `gamma_reg` during an active batch updates
`gamma_int` for the next batch but does not affect the current one.

## Diagnostic Error Codes

| Port | Width | Behaviour |
|------|-------|-----------|
| `error` | 1-bit | Asserts high on first fault; holds until `rst_n` |
| `error_code` | 4-bit | Latches code of first fault; holds until `rst_n` |

Both are **sticky** — cleared only by asserting `rst_n` low for ≥ 4 clock cycles.

| Code | Constant | Priority | Condition | MCU action |
|------|----------|----------|-----------|------------|
| `0x0` | `ERR_NONE` | — | No fault | — |
| `0x1` | `ERR_SV_ZERO` | Highest | `Σ sv_count = 0` when FSM leaves IDLE | Re-program `num_sv_per_class`, assert `rst_n` |
| `0x2` | `ERR_SV_OVERFLOW` | ↑ | `Σ sv_count > NUM_SV` | Reduce per-class SV counts, assert `rst_n` |
| `0x3` | `ERR_ILLEGAL_STATE` | ↑ | FSM reached `default` branch | Assert `rst_n`; inspect waveform |
| `0x4` | `ERR_GAMMA_SAT` | ↓ | `gamma_int > 0x2000` (γ > 8.0) | Reprogram `gamma_reg` to ≤ `0x2000`, assert `rst_n` |
| `0x5` | `ERR_FIFO_OVERFLOW` | ↓ | `fifo_full && qspi_valid` during LOAD_FIFO | Reduce QSPI clock rate or increase `FIFO_DEPTH`, assert `rst_n` |
| `0x6` | `ERR_GAMMA_ZERO` | Lowest | `gamma_int == 0` while FSM not idle — all kernels silently return 1.0 | Reprogram `gamma_reg` to a non-zero value, assert `rst_n` |
| `0x7–0xF` | *(reserved)* | — | Reserved for future extension | — |

## Power Budget (Wearable Strap-On Detector, 180 nm ASIC)

### Operating Modes

**Monitor Mode** — continuous (~99.8% of the time). ECG sampled at 360 Hz;
MCU runs R-peak detection; SVM core is clock-gated.

**Compute Mode** — burst per heartbeat (~857 ms at 70 bpm). Duration:
256 µs QSPI + ~1.3 ms SVM (236 SVs × 275 cycles @ 50 MHz) + ~50 µs OvO ≈ **1.6 ms/beat**.
Duty cycle: 1.6 ms / 857 ms ≈ **0.19%**.

### Per-Component Power

| Component | Monitor | Compute | Notes |
|-----------|---------|---------|-------|
| ECG analog front end | 1.0 mW | 1.0 mW | 300 µA @ 3.3 V |
| Host MCU | 0.5 mW | 3.0 mW | 4 MHz idle vs. active OvO |
| SVM ASIC core | 0.5 mW | 10.0 mW | Clock-gated vs. 50 MHz active |
| Off-chip SRAM | 0.3 mW | 3.3 mW | Standby vs. burst |
| **Subtotal** | **2.3 mW** | **17.3 mW** | |
| BLE radio *(optional)* | +0.5 mW | +0.5 mW | Alert-only advertising |

**Average:** P_avg ≈ 2.33 mW (2.83 mW with BLE) → **0.74 mA** battery draw
(0.90 mA with BLE) at 3.7 V / 85% regulator efficiency.

### Battery Sizing

| Battery | Capacity | Without BLE | With BLE |
|---------|----------|-------------|----------|
| CR2032 | 240 mAh | ~13.5 days | ~11 days |
| 150 mAh LiPo | 150 mAh | ~8.5 days | ~7 days |
| 200 mAh LiPo | 200 mAh | ~11.2 days | ~9.3 days |

## Multi-Scale Feature Vector (256 dims, Q6.10)

| Dims | Source | How computed |
|------|--------|-------------|
| `[0:128]` | Single-beat morphology | 128 samples centred on R-peak, amplitude-normalised to [−1, 1] |
| `[128:192]` | 10-beat mean morphology | Mean of 10 × 64-sample snippets, amplitude-normalised to [−1, 1] |
| `[192:256]` | 100-beat RR track (causal) | 99 RR intervals ÷ NORMAL_RR (308), clipped to [0, 2], resampled to 64 points |

The RR track is causal — no lookahead beyond a 100-entry ring buffer of R-peak timestamps (~200 B).
